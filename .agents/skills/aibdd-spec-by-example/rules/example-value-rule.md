# Example 值約束

spec-by-example 產出的 Example，其具體值須滿足下列約束。

## 約束

1. 可讀成數字的欄位型別須一致
   1. Example 中值本身是數字、或可被解析為數字的字串（如 `1000`、`3`、`001`、`0911111111`，含前導零者，不論金額／數量／等級／代碼／電話），其「字串 vs 數值」型別須全 spec 一致，且反映團隊的統一約定。
   2. 例：會員等級 `3`、金額 `1000`、手機號 `0911111111` 都能讀成數字，型別須一致。

2. 具體值須以 spec 原文為準
   1. Example 的具體範例值與配分、區間、門檻等數字一律取自 spec 原文，不得自行編造。
   2. 例：spec 寫「滿千折百」，Example 的門檻只能是 `1000`、折抵只能是 `100`，不得自訂 `500` 折 `50`。
